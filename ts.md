ТЕХНИЧЕСКОЕ ЗАДАНИЕ
Назначение: Интеграция ИИ-ассистента в существующее ПО с технологией виртуальной реальности 
Область применения: Школьные лабораторные работы (Физика, Химия, Биология)
1. Цель и назначение системы
Цель проекта — интеграция интерактивного ИИ-помощника в существующее программное обеспечение с технологией виртуальной реальности. Ассистент должен функционировать внутри виртуального пространства как умный гид, помогающий ученику в двух направлениях:
Контекст сценария: Ответы на вопросы «Что делать дальше?», «Как выполнить это действие?», «Зачем мы проводим этот этап?».
Предметная область: Разъяснение теоретического материала по физике, химии и биологии в рамках текущей лабораторной работы.
2. Архитектура и интеграция API
Базовый стек: Использовать API-интерфейс, аналогичный архитектуре OpenAI (ChatGPT API) (например, интеграция через официальный SDK или создание аналогичного прокси-клиента, поддерживающего методы chat/completions).
Исключить разработку собственного серверного бэкенда. Система должна работать напрямую через сторонние готовые облачные сервисы без поднятия выделенных серверов со стороны Заказчика.
Связь: VR-программа (клиент) должна напрямую отправлять запросы через интернет к официальным эндпоинтам (URL) выбранного ИИ-провайдера, используя предоставленный API-ключ.
Контекст и База знаний: База знаний (школьные материалы) должна быть загружена в облачное хранилище ИИ-провайдера (например, через механизмы OpenAI Assistants API / Vector Store), чтобы ИИ обращался к ним в облаке, а не на локальном компьютере.
3. База знаний и обучение ИИ (RAG-архитектура)
Для исключения «галлюцинаций» ИИ должен отвечать строго в рамках предоставленной базы данных. Разработчику необходимо реализовать систему RAG (Retrieval-Augmented Generation).
3.1. Загрузка школьных материалов (Предметная область)
Входные данные: Заказчик предоставляет файлы (PDF, DOCX) с учебными материалами, методичками и теоретической базой по физике, химии и биологии.
Задача разработчика: Настроить парсинг, эмбеддинг (векторизацию) этих материалов и сохранение в векторную базу данных для оперативного поиска ИИ перед генерацией ответа.
3.2. Загрузка логики сценариев (Контекст ПО)
Входные данные: Текстовое описание скриптов, шагов, логики и целевых действий каждой лабораторной работы, зашитых в ПО.
Задача разработчика: ИИ должен «знать» архитектуру текущей сцены. При отправке запроса из ПО в ИИ должен передаваться текущий статус: какой шаг выполняет юзер, какие предметы у него в руках, какая это лабораторная работа.
4. Ссылки на источники информации (Прозрачность ИИ)
ИИ должен не просто выдавать ответ, но и подтверждать его достоверность.
Требование: После формирования ответа на основе базы знаний, ИИ обязан указать первоисточник (например: «Источник: Учебник Физики 8 класс, Глава 3, стр. 14» или «Источник: Инструкция к лабораторной работе №2»).
Альтернативный сценарий: Если ответ объемный, ИИ в конце фразы должен предложить опцию: «Хочешь узнать, из какого учебника/документа эта информация?» и при подтверждении выводить источник.
5. Голосовое взаимодействие (STT и TTS)
Интерфейс взаимодействия внутри виртуального пространства должен быть максимально бесшовным и не требовать постоянного ввода текста с клавиатуры.
5.1. Распознавание речи (Speech-to-Text — STT)
Функционал: Ученик зажимает кнопку на контроллере и тем самым активирует распознавание голоса -> ПО записывает аудио с микрофона шлема -> Аудио конвертируется в текст -> Текст отправляется в ИИ.
5.2. Синтез речи (Text-to-Speech — TTS)
Функционал: Полученный от ИИ текстовый ответ конвертируется в аудиопоток в реальном времени.
Требование к звуку: Голос должен звучать естественно (без выраженного роботизированного эффекта), подходить под роль школьного учителя/помощника. Звук должен быть интегрирован в аудио-движок ПО (желательно с поддержкой пространственного 3D-звука, если у ассистента есть виртуальный аватар).
6. Логика работы UI/UX (Как это выглядит для пользователя)
Активация: Пользователь вызывает ИИ (кликом на виртуального робота/помощника - 3д персонажа, нажимая по кнопке на контроллере - возможно рядом с персонажем будет кнопка Задать вопрос ИИ).
Запрос: Ученик задает вопрос голосом: «Зачем мы сейчас нагреваем эту пробирку?» или «Что такое валентность?».
Отображение:
На виртуальном экране рядом с персонажем появляется текстовая расшифровка его вопроса (для контроля распознавания).
Появляется анимированный индикатор «ИИ думает».
Выводится текстовый ответ ИИ + параллельно включается озвучка голосом.
Внизу текста отображается блок: «Источник: [Название документа/Параграф]».
7. Критерии приемки работы
ИИ корректно отвечает на вопросы по физике, химии и биологии, используя только загруженные файлы, и не придумывает факты от себя.
ИИ безошибочно подсказывает текущий шаг в лабораторной работе, ориентируясь на переданное состояние ПО.
Время задержки от окончания фразы пользователя до начала ответа ИИ (включая генерацию голоса) не превышает 5 секунд.
Функция генерации ссылок на источники работает стабильно для каждого ответа, базирующегося на учебных материалах.
